# Guidelines for Git Commit Messages and Dry Run Files

- **Keep git commit messages short and concise.**  
  Use the word **"problem"** instead of "implementation" in commit messages.  
  _Example:_

  ```
  add greedy algorithm problem
  ```

  _(not "implement greedy algorithm")_

- **Refer to dry runs in:** `src/01-warm-up/05_second-largest-dry-run.md`.**Include only a single test case dry run per dry run file.**
